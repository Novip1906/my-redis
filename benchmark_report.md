# Отчет о бенчмарке форматов AOF (Append Only File)

## Среда тестирования

| Параметр | Значение |
| :--- | :--- |
| CPU | Apple M1 Pro |
| RAM | 16 GB |
| OS | macOS 26.3.1 |
| Go | 1.25.1 darwin/arm64 |

## Методология

Бенчмарк выполняет **последовательную запись 1 000 000** команд `SET my_test_key value` в файл для каждого из 4 форматов сериализации. Запись идёт синхронно, без конкурентности, чтобы измерить чистую производительность сериализации и записи в файл. По окончании записи вызывается `file.Sync()` для гарантии сброса буферов на диск. Результаты усреднены по 3 запускам.

## Анализ форматов

### 1. Text (Простой текст)
*   **Плюсы:** Минимальный размер файла (только команда + `\n`). Легко отлаживать: формат отлично подходит для разработки, так как файл легко прочитать и изменить вручную без специальных утилит.
*   **Минусы:** Сложно парсить, если значения ключей содержат спецсимволы или пробелы (требуется экранирование). При чтении необходимо сканировать до разделителя `\n`.

### 2. Binary (Бинарный)
*   **Плюсы:** Самый быстрый на запись. Быстрое чтение благодаря length-prefix: первые 4 байта указывают длину записи, что позволяет сразу считать нужное количество байт без сканирования разделителей.
*   **Минусы:** Данные нечитаемы для человека без специальных инструментов.

### 3. JSON
*   **Плюсы:** Универсальность, читаемость, широкая поддержка инструментов.
*   **Минусы:** Медленнее текстового и бинарного форматов из-за рефлексии при `json.Marshal`. Увеличенный размер файла за счёт ключей объектов.

### 4. RESP (Redis Serialization Protocol)
*   **Плюсы:** Стандарт для Redis. Позволяет использовать существующие инструменты парсинга и обеспечивать совместимость.
*   **Минусы:** Самый медленный и самый большой размер файла (из-за обилия служебных символов `\r\n`, `$`, `*`).

## Итоговая рекомендация
Для production рекомендуется **RESP** (для совместимости с экосистемой Redis) или **бинарный формат** (для максимальной производительности: наибыстрейшая запись и O(1) чтение при восстановлении из AOF). Текстовый формат идеально подходит для этапа разработки и отладки благодаря минимальному размеру и человекочитаемости.
